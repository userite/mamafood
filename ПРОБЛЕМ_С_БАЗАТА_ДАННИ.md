# Проблем: ENOTFOUND - Hostname не може да бъде намерен

## Проблем
Сървърът не може да се свърже с базата данни. Грешката показва:
```
Error: getaddrinfo ENOTFOUND dpg-d426a3p5pdvs73f0ifk0-a
```

## Причини

### 1. Базата данни в Render.com е спряна (най-често)
Render.com free tier базите данни се **спират автоматично** след период на неактивност.

**Решение:**
1. Влез в [Render Dashboard](https://dashboard.render.com)
2. Отиди на твоята PostgreSQL база данни
3. Ако е спряна, ще видиш бутон "Resume" или "Start"
4. Кликни на него, за да стартираш базата
5. Изчакай 1-2 минути докато се стартира
6. Вземи новия connection string (може да е различен след рестарт)

### 2. Базата данни е изтрита
Ако базата данни не съществува в Render dashboard.

**Решение:**
Създай нова PostgreSQL база в Render.com:
1. Влез в Render Dashboard
2. "New +" → "PostgreSQL"
3. Попълни детайлите
4. Вземи connection string
5. Обнови `.env` файла

### 3. Connection string е неправилен
Провери дали connection string-ът в `.env` файла е правилен.

**Решение:**
1. Отвори `.env` файла
2. Провери DATABASE_URL
3. Форматът трябва да е: `postgresql://username:password@host:port/database`
4. Вземи нов connection string от Render dashboard

## Алтернатива: Използвай локална PostgreSQL

Ако не искаш да използваш Render.com, можеш да използваш локална PostgreSQL:

### Стъпка 1: Инсталирай PostgreSQL
- [Download PostgreSQL](https://www.postgresql.org/download/windows/)

### Стъпка 2: Създай база данни
```sql
CREATE DATABASE mamafood;
```

### Стъпка 3: Обнови .env файла
```
DATABASE_URL=postgresql://postgres:твоята_парола@localhost:5432/mamafood
```

## Проверка

След като коригираш проблема, стартирай:
```bash
test-database.bat
```

Това ще провери дали връзката работи.

## Важно за Render.com Free Tier

- Базите данни се спират след 90 дни неактивност
- След спиране, трябва да ги стартираш ръчно
- Connection string-ът може да се промени след рестарт
- Винаги вземи най-новия connection string от dashboard
